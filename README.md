# 学生流失预警与干预分析

## 项目来源

数据来源为学生花名册类业务数据，字段覆盖班级、科目、年级、互动行为、缴费课次、续读状态等信息。由于原始明细包含学员编码和业务敏感字段，本发布版不公开原始 Excel、完整学生明细、Top50 干预名单，仅保留聚合结果表、图表、SQL、Notebook 和分析代码。

| 项目项 | 内容 |
|---|---|
| 数据主题 | 学生续读风险识别与干预优先级分析 |
| 样本量 | 5,527 |
| 流失样本数 | 756 |
| 整体流失率 | 13.68% |
| 输出形式 | 风险等级、原因标签、建议动作、聚合图表 |

## 项目背景

教育机构在续读运营中，既要尽早识别高风险学生，也要把有限的人力优先投入“更容易挽回、且更值得挽回”的对象。本项目将“下季度是否继续在读”转化为二分类问题，并在模型输出后进一步补充风险等级、流失原因标签和干预优先级分数，帮助老师与运营人员更高效地安排跟进动作。

## 分析思路

1. 读取学生花名册，明确目标变量为“下季度常规是否在读”。
2. 清理明显结果型字段和可能造成数据泄露的字段，保留可用于预测的行为与业务特征。
3. 使用随机森林模型输出学生流失概率，并转换为高、中、低风险等级。
4. 基于互动、服务、缴费和新老生特征构建流失原因标签。
5. 设计 `干预优先级分数 = 流失概率 × 可挽回系数 × 学员价值系数`，把模型结果转成业务跟进顺序。
6. 公开输出聚合结果表和图表，不公开个人敏感明细。

## 关键结果摘录

### 模型总览

| 指标 | 数值 | 说明 |
|---|---:|---|
| 样本量 | 5,527 | 参与建模和输出预警的学生记录数 |
| 流失样本数 | 756 | 下季度常规是否在读=否 |
| 流失率 | 13.68% | 流失样本占总样本比例 |
| Accuracy | 0.7055 | 整体预测准确率 |
| Precision | 0.2820 | 预警为流失的人中真实流失的比例 |
| Recall | 0.7460 | 真实流失学生中被识别出来的比例 |
| AUC | 0.8031 | 模型区分能力 |

### 风险等级命中情况

| 风险等级 | 合计 | 真实流失数 | 真实未流失数 | 真实流失占比 |
|---|---:|---:|---:|---:|
| 高风险 | 303 | 222 | 81 | 73.27% |
| 中风险 | 1,768 | 376 | 1,392 | 21.27% |
| 低风险 | 3,456 | 158 | 3,298 | 4.57% |

### 原因标签与动作节选

| 观察点 | 结果 |
|---|---|
| 最主要原因标签 | 互动弱风险，占比 78.98% |
| 次要原因标签 | 互动弱风险、购买意愿弱风险，占比 12.34% |
| 最常见动作 | 正常维护，持续观察互动和缴费变化，占比 62.53% |
| 重点触达动作 | 当天电话/微信重点触达，补充学习反馈并确认续读意向，占比 4.34% |

## 主要结论

- 高风险学生只占整体样本的 5.48%，但其中真实流失占比达到 73.27%，适合作为重点跟进名单。
- 模型召回率为 0.7460，说明大部分真实流失学生能够被提前识别出来。
- 风险原因高度集中在互动弱风险，说明互动频率和沟通质量是续读运营中的关键先行信号。
- 只按流失概率排序不够，叠加可挽回系数和学员价值系数后，更适合指导老师和课程顾问安排优先级。
- 出于公开仓库数据安全考虑，本项目只展示聚合结果，不公开原始 Excel、个人明细和敏感业务名单。
